Apa itu Deployment?

Deployment adalah cara mengelola Pod dengan fitur:

  • Replicas - Menjalankan multiple Pod
  • Self-healing - Otomatis recreate Pod yang crash
  • Rolling update - Update tanpa downtime
  • Rollback - Kembali ke versi sebelumnya

Praktikum

Praktik 1: Membuat Deployment

Struktur folder:

ğŸ“ kubernetes-praktik/
  â”œâ”€â”€ ğŸ“„ deployment.yaml
  â””â”€â”€ ğŸ“„ service.yaml
deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3                   # Jumlah Pod
  selector:
    matchLabels:
      app: nginx                 # Label untuk identify Pods
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
Terminal
# Apply Deployment
kubectl apply -f deployment.yaml

# Lihat Deployment
kubectl get deployments

# Lihat Pods yang dibuat
kubectl get pods

# Lihat detail Deployment
kubectl describe deployment nginx-deployment

Praktik 2: Membuat Service NodePort

service.yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: NodePort
  selector:
    app: nginx                    # Harus match dengan label Pod
  ports:
  - port: 80                      # Port di dalam cluster
    targetPort: 80                # Port container
    nodePort: 30080               # Port untuk akses dari luar (30000-32767)
Terminal
# Apply Service
kubectl apply -f service.yaml

# Lihat Service
kubectl get services

# Akses via Minikube
minikube service nginx-service

Berhasil!

Browser akan terbuka dengan halaman nginx!

Praktik 3: Self-Healing

Terminal
# Lihat Pods
kubectl get pods

# Hapus salah satu Pod
kubectl delete pod [nama-pod]

# Lihat lagi - Pod baru otomatis dibuat!
kubectl get pods

# Jumlah tetap 3 karena replicas: 3

Praktik 4: Scaling

Terminal
# Scale up ke 5 replicas
kubectl scale deployment nginx-deployment --replicas=5

# Lihat Pods bertambah
kubectl get pods

# Scale down ke 2
kubectl scale deployment nginx-deployment --replicas=2

# Lihat Pods berkurang
kubectl get pods

Praktik 5: Update Image (Rolling Update)

Terminal
# Update image ke versi lain
kubectl set image deployment/nginx-deployment nginx=nginx:alpine

# Lihat proses rolling update
kubectl rollout status deployment/nginx-deployment

# Lihat history
kubectl rollout history deployment/nginx-deployment

# Rollback ke versi sebelumnya
kubectl rollout undo deployment/nginx-deployment

Cleanup

Terminal
# Hapus semua resource dari file
kubectl delete -f deployment.yaml
kubectl delete -f service.yaml

# Atau hapus semua sekaligus
kubectl delete all --all

Tugas Praktikum

Tugas: Deploy Aplikasi Flask 100 poin

  1. Gunakan image Flask yang sudah di-push ke Docker Hub di pertemuan 5
  2. Buat file deployment.yaml dengan:
    • replicas: 3
    • image: [username]/flask-app:1.0
  3. Buat file service.yaml dengan tipe NodePort
  4. Deploy ke Minikube
  5. Test self-healing dengan delete Pod
  6. Test scaling (scale ke 5, lalu ke 2)
  7. Screenshot semua hasil dan jelaskan prosesnya
