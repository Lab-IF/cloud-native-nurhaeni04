⚙️ Pengenalan Kubernetes

Mengenal platform container orchestration paling populer

Tujuan Pembelajaran

Memahami apa itu Kubernetes
Menginstall Minikube
Menjalankan Pod pertama
Menggunakan kubectl dasar

Teori

Apa itu Kubernetes (K8s)?

Kubernetes adalah platform open-source untuk mengelola containerized applications secara otomatis.

Analogi Sederhana

Docker = Membuat dan menjalankan container (seperti membuat kapal)
Kubernetes = Mengelola banyak container sekaligus (seperti pelabuhan yang mengatur banyak kapal)

Komponen Utama Kubernetes

Pod

Unit terkecil di K8s. Berisi 1 atau lebih container.

Node

Mesin (physical/virtual) yang menjalankan Pod.

Cluster

Kumpulan Node yang dikelola Kubernetes.

kubectl

CLI tool untuk berinteraksi dengan K8s.

Docker Compose vs Kubernetes

Docker Compose

  • Untuk development
  • Single host
  • Simpel untuk dipelajari
  • File: docker-compose.yml

Kubernetes

  • Untuk production
  • Multiple hosts (cluster)
  • Auto-scaling, self-healing
  • File: deployment.yaml, service.yaml

Praktikum

Praktik 1: Install Minikube

Minikube adalah tool untuk menjalankan Kubernetes lokal di komputer Anda.

Linux/macOS
# Install Minikube (Linux)
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# Atau dengan Homebrew (macOS)
brew install minikube
Start Minikube
# Start cluster
minikube start

# Cek status
minikube status

# Cek versi kubectl
kubectl version --short

Praktik 2: Jalankan Pod Pertama

Terminal
# Jalankan nginx sebagai Pod
kubectl run nginx-pod --image=nginx

# Lihat Pod yang berjalan
kubectl get pods

# Lihat detail Pod
kubectl describe pod nginx-pod

# Lihat logs
kubectl logs nginx-pod

Praktik 3: Perintah kubectl Dasar

PerintahFungsi
kubectl get podsLihat semua Pod
kubectl get pods -o wideLihat Pod dengan detail lebih
kubectl describe pod [nama]Detail spesifik Pod
kubectl logs [nama]Lihat logs Pod
kubectl exec -it [nama] -- bashMasuk ke dalam Pod
kubectl delete pod [nama]Hapus Pod
kubectl get nodesLihat semua Node
Coba Perintah
# Lihat semua resource
kubectl get all

# Masuk ke dalam Pod nginx
kubectl exec -it nginx-pod -- bash

# Di dalam Pod, coba curl localhost
curl localhost
exit

# Hapus Pod
kubectl delete pod nginx-pod

Praktik 4: Dashboard Kubernetes

Terminal
# Buka Kubernetes Dashboard
minikube dashboard

# Akan membuka browser dengan dashboard GUI

Dashboard

Dashboard menyediakan tampilan visual untuk melihat dan mengelola cluster Kubernetes.
